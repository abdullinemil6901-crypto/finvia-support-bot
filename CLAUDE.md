# Support Bot — Finavia

## Описание проекта

Telegram-бот для саппорт-команды Finavia. **30 команд трейдеров**, каждая — отдельная группа в Telegram. Бот автоматически определяет команду по названию чата, создаёт тикеты, уведомляет саппортов и следит за SLA (15 минут ответ).

**Главная цель:** Создать систему, которую могут развивать другие технари. Простой, масштабируемый, надёжный код.

## Жизненный цикл тикета

```
1. Трейдер → /start → выбирает категорию
2. Вводит Order ID или описание
3. БОТ ОПРЕДЕЛЯЕТ КОМАНДУ:
   - message.chat.title → "309 арс"
   - Сохраняет в team_name
4. handlers/actions.py → _send_to_support():
   - Валидация
   - save_ticket() → БД (с team_name и chat_id)
   - send_with_retry() → в саппорт-чат
   - Кнопки "Взял в работу" / "Завершил"
5. Саппорт → клик "Взял в работу":
   - take_ticket() → БД
   - Уведомление в чат трейдеров
6. Саппорт → клик "Завершил":
   - close_ticket() → БД
   - Уведомление в чат трейдеров
```

### Авторассылка при смене дежурных

```
Админ нажимает "Сменить смену" (day → night)
↓
POST /api/duty/switch
↓
Бот получает список chats
↓
Рассылает во все чаты:
"🌙 Ночная смена началась. Дежурит: @username"
```

### Алерт 12 минут (SLA)

```
pg_cron (каждые 2 минуты)
↓
Edge Function: check_stale_tickets()
↓
SELECT * FROM tickets
WHERE status = 'open'
  AND alert_sent = FALSE
  AND created_at < NOW() - INTERVAL '12 minutes'
↓
Если есть:
- Тегает дежурного в саппорт-чате: "@support, тикет #123 не взят!"
- alert_sent = TRUE
```

## Правила работы

### Принципы
- **Простой код** — другие технари должны понимать
- **Минимум зависимостей** — добавлять только когда нужно
- **Всё в одном репо** — легко деплоить

### НЕ делать
- ❌ `git add -A` — только конкретные файлы
- ❌ Хардкодить ID чатов
- ❌ Добавлять функционал сверх задачи
- ❌ Сложные абстракции без необходимости

### После каждой задачи
1. Проверить что ничего не сломалось
2. Запустить тесты
3. Обновить план (отмечать ✅)
