# Support Bot — Полный план проекта

## Описание
Telegram-бот для саппорт-команды. Логирует обращения по типам, строит графики, показывает расписание дежурных, уведомляет трейдеров при закрытии тикета.

---

## Сессия разработки — лог прогресса

### Сессия 1 (2026-06-08)
**Этап:** Начальная разработка  
**Реализовано:**
- Базовая архитектура бота (bot.py, config, database, handlers, keyboards, states, charts)
- 12 категорий обращений через FSM
- Отправка тикетов в чат поддержки
- Уведомление трейдера при закрытии тикета
- /report, /duty, /set_duty
- Статистика по саппортам

**Проблемы/замечания:**
- Хардкод SUPPORT_CHAT_ID в actions.py (нужно вынести в config)
- MemoryStorage теряет FSM-состояния при рестарте бота
- Нет логирования ошибок
- Нет тестов
- SQLite не подходит для продакшена под нагрузкой

## Текущее состояние (готово ✅)

### Архитектура
- `bot.py` — точка входа, polling
- `config/__init__.py` — конфигурация через .env
- `database.py` — SQLite: тикеты, саппорты, расписание
- `handlers/menu.py` — меню, команды /start, /report, /duty, /set_duty
- `handlers/actions.py` — FSM для создания тикетов, взятие/закрытие тикетов + уведомление трейдера
- `handlers/__init__.py` — экспорт роутеров
- `keyboards.py` — inline-клавиатуры
- `states.py` — FSM-состояния для 12 типов обращений
- `charts.py` — генерация 3 графиков через matplotlib (по часам, по типам, по саппортам)
- `schedule_manager.py` — работа с schedule.json (дежурства)

### Функционал
- ✅ 12 категорий обращений (с Order ID и без)
- ✅ FSM для пошагового ввода данных трейдером
- ✅ Отправка тикета в чат поддержки с кнопками "Взял в работу" / "Завершил"
- ✅ Взятие тикета в работу (смена статуса, запись саппорта)
- ✅ Закрытие тикета + **уведомление трейдера** ("Ваш запрос закрыт")
- ✅ /report — 3 графика за сегодня (по часам, по типам, по саппортам)
- ✅ /duty — кто дежурит сейчас (день/ночь МСК)
- ✅ /set_duty — админы обновляют расписание
- ✅ Статистика по саппортам (взято/закрыто/ср. время)

### БД (support_bot.db)
- `tickets` — id, trader_id, username, name, label, order_id, status, taken_by, taken_by_id, taken_at, closed_at, created_at
- `supports` — id, tg_id, username, full_name, added_at
- `schedules` — id, support_id, day_of_week, hour_start, hour_end

## План развития (приоритеты)

### 1. Надёжность и качество (High)
- [ ] **Логирование ошибок** — добавить `logging` в bot.py, handlers, database
- [ ] **Graceful shutdown** — обработать SIGINT/SIGTERM, корректно закрывать polling и БД
- [ ] **Retry при отправке сообщений** — tenacity/exponential backoff для bot.send_message
- [ ] **Валидация входящих данных** — проверить order_id, длину текста, escape HTML
- [ ] **Тесты** — pytest: database (CRUD), handlers (моки), charts (снимки)

### 2. UX / Удобство (Medium)
- [ ] **Пагинация в /report** — если графиков много, отправлять медиа-группой или по кнопкам
- [ ] **Команда /my_tickets** — трейдер видит свои открытые/закрытые тикеты
- [ ] **Команда /stats** — персональная статистика саппорта
- [ ] **Реакция на "тикет взят" у трейдера** — уведомить: "Ваш запрос взял @support"
- [ ] **Экспорт отчёта в Excel/CSV** — для админов

### 3. Админка и управление (Medium)
- [ ] **/add_support, /remove_support** — управление саппортами через бота
- [ ] **/list_supports** — список всех саппортов
- [ ] **Редактирование расписания через кнопки** — не только /set_duty текстом
- [ ] **Роли** — админ / саппорт / трейдер (разные права)

### 4. Масштабируемость (Low → Medium при росте)
- [ ] **Миграция на PostgreSQL (Supabase)** — при нагрузке >10k тикетов/мес
- [ ] **Redis для FSM** — MemoryStorage не выдерживает рестарты
- [ ] **Webhook вместо polling** — для продакшена на сервере
- [ ] **Очередь задач (Celery/arq)** — тяжелые отчёты в фоне

### 5. Мониторинг (Low)
- [ ] **Healthcheck endpoint** — для uptime-монитора
- [ ] **Метрики (Prometheus)** — запросы, ошибки, latency
- [ ] **Sentry / логирование ошибок в Telegram-чат**

## Известные ограничения / технический долг

| Проблема | Решение |
|----------|---------|
| SQLite — файловый, не для продакшена под нагрузкой | Миграция на PostgreSQL |
| MemoryStorage — теряет состояние при рестарте | RedisStorage |
| Polling — не для продакшена | Webhook + nginx |
| Нет тестов | Добавить pytest |
| Нет миграций БД | Alembic при переходе на PG |
| Хардкод SUPPORT_CHAT_ID в actions.py | Вынести в config |
